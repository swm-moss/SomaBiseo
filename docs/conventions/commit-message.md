# 커밋 메시지 컨벤션

SomaBiseo에서 커밋을 만들 때는 이 문서를 따른다. 에이전트가 커밋을 수행하는 경우에도 동일하게 적용한다.

## 1. 기본 형식

```txt
type: 제목

- 변경한 내용
- 변경한 이유
```

- `type`은 영어 소문자로 작성한다.
- 제목과 본문은 한글로 작성한다.
- 제목과 본문 사이에는 빈 행을 둔다.
- 제목 첫 글자는 대문자로 작성한다.
- 제목 끝에는 마침표를 붙이지 않는다.
- 제목은 영문 기준 50자 이내를 목표로 한다.
- 본문에는 어떻게보다 무엇을 왜 바꿨는지 적는다.
- 여러 항목이 있으면 글머리 기호를 사용한다.

## 2. 커밋 유형

| 커밋 유형 | 의미 |
| --- | --- |
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `docs` | 문서 수정 |
| `style` | 코드 formatting, 세미콜론 누락, 코드 자체의 변경이 없는 경우 |
| `refactor` | 코드 리팩토링 |
| `test` | 테스트 코드, 리팩토링 테스트 코드 추가 |
| `chore` | 패키지 매니저 수정, 그 외 기타 수정 ex) `.gitignore` |
| `design` | CSS 등 사용자 UI 디자인 변경 |
| `comment` | 필요한 주석 추가 및 변경 |
| `rename` | 파일 또는 폴더 명을 수정하거나 옮기는 작업만인 경우 |
| `remove` | 파일을 삭제하는 작업만 수행한 경우 |
| `!BREAKING CHANGE` | 커다란 API 변경의 경우 |
| `!HOTFIX` | 급하게 치명적인 버그를 고쳐야 하는 경우 |

## 3. 좋은 예시

```txt
design: 브랜드 로고와 파비콘 적용

- 서비스 워드마크를 로그인 화면과 헤더에 적용
- 작은 앱 아이콘을 favicon, app icon, 헤더 아이콘으로 추가
- 브랜드 자산이 일관되게 노출되도록 공용 public 경로에 정리
```

```txt
docs: 커밋 메시지 컨벤션 문서화

- 에이전트가 참고할 커밋 유형과 작성 규칙 추가
- 커밋 제목과 본문 작성 기준을 AGENTS 문서에서 연결
```

## 4. 피할 예시

```txt
feat: update
```

무엇이 바뀌었는지 알 수 없다.

```txt
Design: 로고 적용.
```

커밋 유형이 대문자이고 제목 끝에 마침표가 있다.

```txt
docs: 커밋 규칙 추가
- 본문과 제목 사이에 빈 행이 없음
```

제목과 본문 사이에는 반드시 빈 행을 둔다.
